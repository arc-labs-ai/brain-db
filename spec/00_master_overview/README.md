## How to read the spec series

The recommended reading order is **not** numerical. Read in this order for a linear introduction:

1. **[`00_index.md`](00_index.md)** — orient yourself in the series.
2. **[01. System Architecture](../01_system_architecture/)** — the conceptual whole. Required before any detail spec.
3. **[02. Data Model](../02_data_model/)** — the entities and their relationships.
4. Detail specs in any order, depending on what you're building or operating.

The numerical order (00 → 16) is the order in which specs are *referenced* and *layered*. The reading order maximizes comprehension.
